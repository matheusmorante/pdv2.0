# Projeto PDV - Regras e Diretrizes

Este arquivo contém as regras e diretrizes que o Antigravity (assistente de IA) deve seguir rigorosamente durante o desenvolvimento deste projeto.

## Regras Globais (USER_GLOBAL)

1.  **Código Limpo e Modularizado:** Siga sempre os princípios de Clean Code e mantenha a estrutura modularizada.
2.  **Registro de Ideias e Planos:** Sempre salve ideias e planos pendentes em um arquivo (Ex: `IDEAS.md` ou `PLANS.md`) para consulta posterior.
3.  **Economia de Cota:** Economize o máximo de cota do plano de assinatura.
4.  **Ambientes (Dev vs Prod):**
    *   Diferencie claramente as configurações para o ambiente de desenvolvimento e produção.
    *   Sempre ajuste rotas, variáveis de ambiente e outros dados ambíguos para funcionar corretamente em ambos.
5.  **Comunicação de Bugs:** Relate imediatamente qualquer bug que esteja impedindo o progresso para que o usuário saiba da demora enquanto o trabalho continua.
6.  **Esclarecimento de Dúvidas:** Pergunte se houver qualquer ambiguidade ou dúvida no entendimento do que foi dito ou mostrado.
7.  **Persistência de Regras:** Salve estas regras sempre em um arquivo no projeto (este arquivo `RULES.md`).

## Fluxos de Trabalho (Workflow)

*   Utilize os comandos de barra (`/create`, `/plan`, `/debug`, etc.) conforme necessário.
*   Para mudanças complexas ou novas funcionalidades, utilize o protocolo de Socratic Gate antes de implementar.

## Design e Estética

*   **Aesthetics Matter:** Designs devem ser modernos, "vibrant", e premium (dark mode, glassmorphism, micro-animations).
*   **Proibido:** Cores genéricas (plain red/blue/green), paletas de cores padrão ou layouts clichês.
*   **Typography:** Use fontes modernas (Inter, Roboto, Outfit).
